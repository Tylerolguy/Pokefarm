Buildings
- a building is a type of item that can be placed
- buildings are crafted via recipes in crafting tables or aquired as rewards
- some buildings start on the map
- buildings may have collosion
- buildings can be stored in chests or inventory
- places a building allows for the building to be constrcuted by bringing pokemon that can construct building
    this is the only skill that does not need pokemon to be in range for (see skilled jobs down below)
- once a building is finished being completed, it can be interacted with to use its functions
- many different categories of buildings, such as resource buildings, bed, and pc
- when talking to a building, ditto icon will appear for the icon, and ponder what should i do with
    this <building name>. with the options depending on the building


Construction
- buildings require to be constructed in order to fully be placed down.
- the default construction skill is construction, but sometimes other skills will also be needed likes
    watering for a fountain
- a construction site has no limit on the amount of pokemon that can work there
- a construction site requires all needed skills to be present before construction can happen
- all pokemon will add there valid skills to the construction effort
- any claimed pokemon can work on the construction regardless of bed zones
- while a pokemon is construction its icon will be on the building and it will be despawned from the map
- the completed effort will be displayed and update as more effort is added
- each building has its own amount of effort needed with better buildings needing more effort typically
- once the building is finished pokemon will be spawned in the exit zone of the construction site
- construction sites have an exit zone like resource buildings


Beds/Claiming pokemon
  - there is a limit to the amount of beds that can be on a farm (raised with player level)
  - a pokemon is unclaimed if it has no bed. it can do anything except be asked to follow to a bed or pc
  - a bed can be assigned to a pokemon if that pokemon likes that type of bed (implement later)
  - While assigned to a bed, a pokemon has a certain range it will wander around in depending on type of bed
      with better beds giving more wander range
  - A bed has a limit to the amount of pokemon that can be assigned there. if trying to assign a pokemon  
      to a full bed, a message informs you that the bed is full.
  - If a bed has room to assign a pokemon it will allow you to assign it and let you know the pokemon 
      lives here
  - Beds are a type of building that have collosion, can be built at a respective crafting table for materials
      with the recipe being more difficult for the status of the bed (holds more pokemon, more range, or
      more skilled pokemon)
  - if a bed is removed from the world, all pokemon that lived there will become unclaimed
  - Pokemon can only have 1 bed, but some beds can have multiple pokemon
  - to unassign a pokemon from the bed, talking to the bed should have the option to unassign the pokemon
      in addition to assigning a pokemon to a new bed will remove it from the old bed and have the new bed
      be its home
  - when talking to a bed, options should include bye, unassign for each pokemon that lives at that location,
      assign <pokemon that is actively following> for each pokemon following. If trying to assign a pokemon
      while the bed has full capacity, it should say, this bed is full. if a pokemon is removed or added
      it should let that player know it happen successfully
  - beds will have a message about who lives at the bed when the player is nearbby to see if beds are occupied



Resource Buildings
 - a resource building is a type of building that produces some sort of resource. there are multiple of resource
    building categories
 - resource buildings require a pokemon to work at the building to produce said resource. different resource
   buildings and resources have different skills required. should only accept valid skills to work
 - each resource has a different effort load, that is required to produce the resource
 - while a pokemon is actively working in the resource building, it adds its respective skill to do effort every 
    second
 - if multiple pokemon are assigned to a resource building, they will all add its respective skill to do the
    effort done.
  - while a pokemon is actively working, they should be despawned off the map
  - when a pokemon switches from actively working to not working, they should spawn/despawn at a exit zone
    which does require non collosion from buildings to ensure pokemon can be spawned without getting stuck
    in buildings
  - to assign a pokemon to a resource building, a pokemon must be following the player and the building
      exit/entrance must be in the pokemon's bed range. 
  - some RB can have multiple assigned pokemon to them, but some can only have 1
  - a pokemon will choose when it works based on its information detailed in the pokmeon section, when it 
      chooses to work it will work until it desires to leave, or until the building has full inventory/
      no work.
  - resources buildings can hold up to a certain amount of produced resources
  - resources buildings should reflect the amount of work progress done on producing a resource and show 
      when the capacity is full
  - when a pokemon is assigned to a building, we should see the icon at the bottom of the building
  - talking to a resource building may have some options detailed in their section, but generally
      it should have options bye, assign pokemon (for each following pokemon), unassign pokemon (for each
      currently assigned), collect resource (which takes all of the produced resources). assigning pokemon
      will fail and give a message for the respective issues, bed is to far, pokemon does not have required 
      skill, pokemon does not have high enough skill level (skills may match but not be the high enough level).
      if none of those objections happen the pokemon should be successfully assigned and be given that message
  - unassign a pokemon via the dialogue option, should safely remove a pokemon from the assigned pokemon. 
      removing its icon from the bottom, and if the pokemon is currently working it is placed at the exit

  - pickup resources (which should be first dialogue option), should remove resources from the inventory of
      the building and add them to the inventory of the player. if the player has a full inventory, it 
      should not do that and instead let the player know that their inventory is full


Farm
 - farm is a type of resource building. there is different levels of farm but they will follow this guideline
 - farms are used to produce berries. 
 - each farm has a different list of berries that can be grown. the player will have to unlock each berry as
    well before it can be grown. (so basic farm can grow oran berry and maybe sitrus berry, but will require
    that player have the sitrus berry recipe to grow that even though the farm works for it)
  - each farm will also be able to have the no berry option.
  - the player will select what berry should be actively grown from a berry menu, with different berries 
      requiring different skills and efforts
  - if no berry is set (which should be the default), then no berry should be growing on the farm
  - if the player changes berries mid effort, it should set the current effort to 0 and the current skill 
      to planting
  - to produce a berry (unless specific cases), the building will have to go through planting effort, then
      watering effort, then harvesting effort. once all 3 have been completed, one berry will be completed.
  - each production stage shoudl have different indicators for what stage of the production we are on
 - farms are required to have minimum of 3 spaces for pokemon to work (unless a very special type of farm)
